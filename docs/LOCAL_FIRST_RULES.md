# Local-First Rules

## Rule 1 — Offline is first-class

SWRLZ must keep useful local behavior when internet, paid APIs, or the online Mentor are unavailable. Local skills, allowlists, logs, context viewing, and safety controls should continue to function.

## Rule 2 — No paid runtime dependency for basic operation

Paid online providers may enhance planning or research, but the local node must not require paid-per-call AI merely to open, view skills, inspect context, run trusted local actions, or save taught deltas.

## Rule 3 — Local learning survives updates

A patch may improve the official core, but it cannot delete local deltas, device profiles, active mission logs, or user safety preferences without explicit user approval.

## Rule 4 — Staging before activation

Updates are downloaded into staging, verified, merged, and smoke-tested before the live runtime changes.

## Rule 5 — User remains sovereign

Medium/high/locked actions require confirmation unless the user intentionally changes trust settings. Emergency pause/takeover must always be available.
